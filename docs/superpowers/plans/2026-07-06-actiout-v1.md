# ActiOut v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Delegation note (user directive):** Each task carries a **Tier** (Opus / Sonnet / Haiku) — dispatch the task to that model with the contract in `.claude/agents/actiout-implementer.md`. The lead agent (Fable) reviews between tasks. Signatures and behavior tables in this plan are binding; implementers write the bodies.

**Goal:** Build ActiOut v1 — a local-first, offline-capable, mobile-first workout tracker (React/TS/Vite/Dexie PWA) per `docs/superpowers/specs/2026-07-06-actiout-v1-design.md`.

**Architecture:** React components read Dexie via `useLiveQuery` and never write directly; pure-TS service modules wrap all writes in Dexie transactions and log `app_events`. Analytics are computed on read. Zustand holds only ephemeral UI state.

**Tech Stack:** React 18, TypeScript strict, Vite, Dexie + dexie-react-hooks, Zustand, react-router-dom, vite-plugin-pwa, Vitest + fake-indexeddb.

## Global Constraints

- TypeScript `strict: true`; `npx tsc --noEmit` must pass at the end of every task.
- Dependencies fixed to: `react`, `react-dom`, `react-router-dom`, `dexie`, `dexie-react-hooks`, `zustand`; dev: `typescript`, `vite`, `@vitejs/plugin-react`, `vite-plugin-pwa`, `vitest`, `fake-indexeddb`. **No additions without lead approval.**
- No UI/component/chart libraries. Hand-rolled CSS with custom properties; charts are hand-built SVG.
- Entity fields are camelCase in code and export JSON (the SQL doc's snake_case is documentation only).
- IDs: `crypto.randomUUID()`. Timestamps: ISO 8601 via `nowIso()`. Calendar dates: local `YYYY-MM-DD` via `todayLocalDate()`. Weekdays: 0–6, 0 = Sunday (matches `Date.getDay()`).
- All stored weights carry a unit snapshot (`'lb' | 'kg'`); conversion happens on read only; factor 1 lb = 0.45359237 kg.
- `sequencePosition` is 1-based, global per session, contiguous after every mutation.
- Analytics read `status === 'completed'` sessions only unless `includeDnf` is passed.
- Session status: `'draft' | 'completed' | 'dnf'`. At most one draft exists at a time (enforced by session-service).
- Copy is quiet and direct; no hype, no emoji in UI text. Tap targets ≥ 44px. Dark-first theming.
- Commit after every task: `git add -A && git commit -m "<type>: <summary>"`.

Behavior (all mutations in one transaction each):
| Rule | Detail |
|---|---|
| Start | If any `status==='draft'` exists → throw `DraftExistsError(draftId)`. | Move | Swap `sequencePosition` with neighbor; no-op at boundary. |
| Add | `ensureExercise`, position = max+1, planned/actual undefined except `weightUnit` = pref unit. |
| Remove | Delete then renumber remaining contiguously 1..n preserving order. |
| Complete/DNF | Only from `draft` (else throw). Set status, `endedAt: nowIso()`, `durationSeconds = round((endedAt-startedAt)/1000)`. |
| Events | `logEvent('session', id, 'started'|'completed'|'dnf'|'item-added'|'item-removed'|'item-moved')`. |
